Playground Features

Your Authipay Developer Playground account provides features for you to test all aspects of your Authipay integration, so you can build the best payment features for your customers.

Check out some of the features:

  • API Simulators
  • API Response Maps & Triggers
  • Integration Guides & Samples
  • Test Reports
  • Live Logs
  • Load Test Simulators
{% include docs/components/text-block-divider.html %}

API Simulators

The Developer Playground has powerful simulators for every Authipay API endpoint and operation. For example, there's a simulator for Authipay Connect, a simulator for 'card-sale', a simulator for 'card-void', etc.

With a simple copy&paste, you can point your development and test environments at your Playground, then start sending test transactions to the simulators - they'll respond as if your code is talkig to the real thing.

And the icing on the cake? Your simulators are just for you (and your team). Unlike traditional sandboxes, your simulators aren't shared with other merchants. Even better, you get to control your own simulator settings such as latency values.

{% include docs/components/text-block-divider.html %}

Response Code Maps & Triggers

Each API simulator has a table of all its possible response codes. The table maps response codes to request values so that you can send in values in your test requests and trigger the response codes you want.

The default trigger parameter is the transaction amount... you pass in different decimal value, you'll get different response codes. You can also use the card CVV value or the Cardholder name as a trigger value - just select the trigger value you want in the Settings of the simulator you're testing.

Say goodbye to crappy test cards. Enjoy testing any response you need. Your imagination is your limit.

Integration Guides

Every simulator comes with integration guides, tips and links to sample apps. The instructions come in multiple coding languagues.

Create your Developer Playground account, open up the API simulator you want, and select the "Integrate" tab to see integration instructions."

We regularly build and update our own example applications. And we believe in sharing what we've learnt. You can check out the code for our examples on our Testing Pays GitHub account.

Test Reports

Each simulator also comes with its own test report. As you and your team hit the simulator with test transactions, the test reports will tell you what you've tested, what you haven't tested, the amount of response codes you've covered

Use the test reports to: firstly capture how much coverage you're achieving with your current test plans, then to drive higher levels of coverage in your testing.

{% include docs/components/text-block-divider.html %}

Live Logs

Live Log gives you a window into each simulator as it processes your test requests. In real-time, you can see each transaction as it hits. The Live Log gives you visibility into how your request is treated, and the response being assembled and sent back. If the simulator finds a problem with your request, you'll also see verbose explanations in the Live Log.

Everything is clearly timestamped so that you can be confident in the simulator roundtrip time for your testing. Simulator response time is consistently under 100ms - unless of course you've activated its 'latency' setting to introduce artificial overhead!

Our Tip: Test Pong

Ever played Pong? Tail your local logs on your dev machine, and in a browser window start the Live Logs on the simulator, then fire off some test requests. Sit back and watch your requests hitting the simulator, and responses bouncing back to your local logs.

Load Test Simulators

Your site should never go down during Cyber Monday, Christmas sales or any big online peak sale event. If you need to test your online store under massive user load, then our load test simulators can support your efforts.

You must book a time slot for our load test simulator. Once we send you connection details, so you can then ramp up your load test scripts. Our load test simulators can accept millions of requests for prolonged periods and handles up to 80 transaction per seconds at a constant 100ms response time.

To make a booking, log into your Developer Playground account, go to your "CURRENT TEAM" menu and complete the simple booking form. The team will respond within an hour.

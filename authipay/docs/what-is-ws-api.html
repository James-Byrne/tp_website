What is the WebServices API?

Authipayâ€™s WebServices API (WS API) is a comprehensive gateway to a wide range of online payments functions. It is an XML-based SOAP API. All examples in the Developer Playground use some sort of XML tree builder.

As a developer, by going the WebServices route, you'll have more coding to do for your integration than by going the Connect API route. On the flip side, WS API allows you to build much more powerful and slick customer payment features. In addition, WS API is the only option if your customer checkout has to be fully branded in keeping with the rest of your site.

The Authipay Developer Playground can help you build these features better and faster. It provides powerful simulators for all core operations on the WS API. Whether you need to build a brilliant checkout flow for your customers, or give your operations team time-saving returns tools, the simulators can help you explore and trial these features.

Web Services API Features

The the Authipay Developer Playground provides simulators for you to test and explore the following WS operations:

  • Card Sale: authorise and capture card payments in one shot.
  • Card Pre-Auth: reserve an amount on card for later capture.
  • Card Post-Auth: capture an amount that was previously reserved on a card.
  • Force Ticket: force a capture on a card.
  • Return: return an amount for a previous order.
  • Credit: refund an amount to a customer card.
  • Void: cancel a transaction that's in progress.
  • Datavault: store a card for use in future transactions.
  • Dynamic Pricing: DCC to give customers the option to pay in their home currency.

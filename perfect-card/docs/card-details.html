Use the card-details sim

Once you created your virtual card using the card-token sim, you can get the card details of this virtual card. We created the card details for you beforehand, and this API sim simply returns them based on the CardId's last 2 digits.

You can however use the sim independently as well, regardless of previous calls and setup.

  1. Use the preconfigured scenarios
  2. Test card-details independently
{% include docs/components/text-block-divider.html %}

1. Use the preconfigured scenarios

These scenarios are described in detail in the Test tab of card-token sim. These scenarios stretch over multiple sims, all starting with creating a virtual card using the card-token sim. The initial call creates the card details, which are remembered for you. In this case however you need to remember to use the CardId returned to you on the very first call.

{% include docs/components/text-block-divider.html %}

2. Test card-details independently

If you are simply writing unit tests, and would like to test the sims independent from each other - you are free to do so! All you need is to change / add 2 digits at the very end of the CardID, or in some sims' case change the trigger parameter. Changing the triggerpart of your request will mean you can invoke the responses you need without any prior setup using the get-token sim.

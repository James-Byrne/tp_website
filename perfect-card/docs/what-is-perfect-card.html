What are the Perfect Card sims?

Perfect Card APIs are a simple way to create a virtual card with a certain amount. For example, each time you call card-token API sim on Testing Pays, we create an individual MasterCard number, and assign it to a CardId. This CardId is returned to you - unless you have chosen to simulate an error, i.e. a System Error, or Invalid Currency code.

Here you can choose a preconfigured scenarion in the Test tab - by which we can guarantee the outcome of the next call(s) you make to other Perfect Card sims. Like an IP Restriction error that fails on card-details. If you would like to do individual sim requests, regardless of the scenarios, just use the dedicated mapping tables of the sims.

Sims Overview

Perfect Card sims are dependent on each other, although they can be used individually. Hereâ€™s how a typical flow would work with them:

  1. Create a new virtual card, using the card-token call, which returns a CardId to be used in later calls.
  2. You supply this CardID along with the CardToken returned from the previous call to get the card's full details using the card-details API call.
  3. You can modify the balance of the card with adjust-card-balance, where the Amount will be either added or deducted from your balance - depends on whether you have sent a positive or negative value in this field.
  4. If the CardToken expired (after 60 seconds), you can request a new one, using your CardId. Just call the card-get-token API.
